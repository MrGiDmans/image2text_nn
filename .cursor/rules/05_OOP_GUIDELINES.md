# Правила ООП для проекта Captioning Model

1. **Модули не должны знать о структуре данных вне своей зоны.**  
   Encoder не знает о словаре, Decoder не знает о размерах изображений.

2. **Модели — закрыты для модификации, открыты для расширения.**  
   Хочешь Transformer — просто создаёшь новый класс, API не меняется.

3. **Композиция.**  
   CaptionModel содержит encoder+decoder, а не наследует.

4. **Каждый класс полностью тестируем.**

5. **Методы имеют четкую ответственность.**  
   Хорошо: `encode(image)`  
   Плохо: `encode_and_save_and_plot(image)`

6. **Конфигурация выносится в отдельный config-файл.**

7. **Для портирования в R отдельные слои должны быть независимыми.**
